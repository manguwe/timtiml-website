What is HTML?

Tags, elements · ~4 min
✓ Done ▼

HTML stands for HyperText Markup Language. It is the standard language used to create web pages. HTML uses tags — special labels wrapped in angle brackets — to tell the browser what each piece of content is.

HTML
<!-- This is a comment — the browser ignores it -->

<!-- A paragraph tag -->
<p>Hello, I am learning HTML with Anotida!</p>

<!-- A heading tag -->
<h1>Welcome to TIMTIML</h1>

<!-- Tags have an OPENING tag and a CLOSING tag (note the slash /) -->
💡

Most HTML tags come in pairs: an opening tag <p> and a closing tag </p>. The content goes between them. The closing tag always has a forward slash /.

Basic HTML Page Structure

doctype, head, body · ~5 min
✓ Done ▼

Every HTML page follows the same skeleton structure. You must always start with this template — it tells the browser what kind of file this is and where to find the title and content.

HTML
<!-- 1. Doctype — tells browser this is HTML5 -->
<!DOCTYPE html>

<!-- 2. The root element -->
<html lang="en">

  <!-- 3. HEAD — invisible info about the page -->
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Website</title>
  </head>

  <!-- 4. BODY — everything visitors SEE -->
  <body>
    <h1>Hello World! 🌍</h1>
    <p>This is my very first webpage.</p>
  </body>

</html>
✅

Try it: Copy this code, paste it into Notepad (or VS Code), save as index.html, then double-click the file. It opens in your browser — your first web page!

03

Text & Headings

h1-h6, p, strong, em · ~5 min
✓ Done ▼

HTML has 6 heading levels and several text formatting tags. Using the right tag matters — search engines and screen readers rely on them.

HTML
<!-- Headings: h1 is the biggest, h6 is smallest -->
<h1>Main Page Title (use only once per page)</h1>
<h2>Section Heading</h2>
<h3>Sub-section Heading</h3>

<!-- Paragraph -->
<p>This is a normal paragraph of text.</p>

<!-- Text formatting -->
<p>
  This word is <strong>bold</strong>.
  This word is <em>italic</em>.
  This is <mark>highlighted</mark>.
  This is <del>strikethrough</del>.
  This is <br> a line break.
</p>

<!-- Horizontal rule (divider line) -->
<hr>
⚡

Only use <h1> once per page — it's the main title. Use <h2> for section headings, <h3> for sub-sections. This helps Google understand your page structure.

04

Links & Images

a href, img src · ~6 min
✓ Done ▼

Links and images are what make the web alive. The <a> tag creates clickable links and the <img> tag displays images. Both use attributes to specify where to go or what to show.

HTML
<!-- Link — href is the destination -->
<a href="https://timtiml-website.vercel.app">Visit TIMTIML</a>

<!-- Opens in a new tab -->
<a href="https://google.com" target="_blank">Google</a>

<!-- Link to another page on your own site -->
<a href="about.html">About Me</a>

<!-- Link to a section on the same page -->
<a href="#contact">Jump to Contact</a>

<!-- Image — src is the image URL, alt is description -->
<img src="profile.jpg" alt="Photo of Anotida" width="200">

<!-- Image from the internet -->
<img src="https://picsum.photos/300/200" alt="Random photo">
💡

Always add an alt attribute to images — it describes the image for blind users and for Google Image Search. Never leave it empty.

05

Lists — Ordered & Unordered

ul, ol, li · ~4 min
✓ Done ▼

HTML has two main types of lists. Unordered lists show bullet points. Ordered lists show numbers. Both use <li> for each item inside.

HTML
<!-- Unordered list (bullet points) -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Ordered list (numbered) -->
<ol>
  <li>Create the HTML file</li>
  <li>Add CSS styling</li>
  <li>Add JavaScript</li>
  <li>Deploy to the web</li>
</ol>

<!-- Nested list (list inside a list) -->
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>

06

Tables — Displaying Data in Rows & Columns

table, tr, td, th · ~6 min
✓ Done ▼

Tables organise data into rows and columns, like a spreadsheet. Use them for comparing data, schedules, or results — not for page layout (CSS handles that).

HTML
<table>

  <!-- thead — table header -->
  <thead>
    <tr>  <!-- tr = table row -->
      <th>Name</th>    <!-- th = header cell (bold) -->
      <th>Language</th>
      <th>Level</th>
    </tr>
  </thead>

  <!-- tbody — table body -->
  <tbody>
    <tr>
      <td>Anotida</td>  <!-- td = normal cell -->
      <td>Python</td>
      <td>Advanced</td>
    </tr>
    <tr>
      <td>Visitor</td>
      <td>HTML</td>
      <td>Beginner</td>
    </tr>
  </tbody>

</table>
⚡

Never use tables for layout (placing columns/sidebars). That was old 1990s HTML. Use CSS Flexbox or Grid for layout instead — tables are only for actual data.

07

Forms — Collecting User Input

input, button, select · ~8 min
✓ Done ▼

Forms let users send data to your website — sign up, log in, send a message, place an order. Every form starts with a <form> tag and contains various input types.

HTML
<form action="submit.php" method="post">

  <!-- Text input -->
  <label for="name">Your Name:</label>
  <input type="text" id="name" placeholder="Enter your name" required>

  <!-- Email input -->
  <label for="email">Email:</label>
  <input type="email" id="email" placeholder="you@example.com">

  <!-- Dropdown -->
  <label for="service">Service Needed:</label>
  <select id="service">
    <option value="design">Graphic Design</option>
    <option value="web">Web Development</option>
  </select>

  <!-- Textarea -->
  <label for="msg">Message:</label>
  <textarea id="msg" rows="4" placeholder="Your message..."></textarea>

  <!-- Submit button -->
  <button type="submit">Send Message 📨</button>

</form>
💡

Always pair <label> with your inputs using for="inputId". This makes your form accessible to screen readers and clicking the label focuses the input.

08

Semantic HTML — Meaningful Tags

header, nav, main, footer · ~6 min
✓ Done ▼

Semantic HTML uses tags that describe what the content IS, not just how it looks. This helps Google understand your page and makes your code professional and accessible.

HTML
<!DOCTYPE html>
<html lang="en">
<head>
  <title>TIMTIML — Anotida Manguwe</title>
</head>
<body>

  <!-- Semantic tags — describe purpose, not style -->

  <header>
    <nav>
      <a href="#home">Home</a>
      <a href="#about">About</a>
    </nav>
  </header>

  <main>
    <section id="home">
      <h1>Welcome to TIMTIML</h1>
      <p>Web developer and graphic designer from Zambia.</p>
    </section>

    <article>
      <h2>Latest Blog Post</h2>
      <p>Content here...</p>
    </article>

    <aside>
      <p>Sidebar content or ads</p>
    </aside>
  </main>

  <footer>
    <p>© 2025 TIMTIML — Anotida Manguwe</p>
  </footer>

</body>
</html>
🏆

You now know the essential HTML! The logical next step is CSS — which controls colours, fonts, spacing and layout. Click "CSS" in the language bar above to continue your journey!
