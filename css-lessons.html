01

What is CSS & How to Add It

inline, internal, external · ~4 min
✓ Done ▼

CSS stands for Cascading Style Sheets. It controls how HTML elements look — colours, sizes, fonts, spacing, layout. There are 3 ways to add CSS to a page.

HTML + CSS
<!-- 1. INLINE — style on one element (avoid this) -->
<p style="color: red; font-size: 20px;">Red text</p>

<!-- 2. INTERNAL — inside <style> in <head> -->
<head>
  <style>
    p { color: blue; }
  </style>
</head>

<!-- 3. EXTERNAL — separate .css file (BEST PRACTICE) -->
<head>
  <link rel="stylesheet" href="style.css">
</head>

/* In style.css */
p {
  color: #00d4ff;
  font-size: 18px;
}
⚡

Always use an external CSS file for real projects. It keeps your HTML clean, lets you reuse styles across pages, and makes maintenance much easier.

02

Selectors — Targeting Elements

tag, class, id, pseudo · ~6 min
✓ Done ▼

CSS selectors tell the browser which HTML element(s) to style. The right selector makes your CSS precise and maintainable.

CSS
/* Tag selector — styles ALL <p> elements */
p { color: white; }

/* Class selector — styles elements with class="card" */
.card { background: #0f1628; border-radius: 10px; }

/* ID selector — styles ONE element with id="hero" */
#hero { text-align: center; }

/* Descendant — <p> inside .card only */
.card p { font-size: 0.9rem; }

/* Multiple selectors */
h1, h2, h3 { font-weight: 900; }

/* Pseudo-class — style on hover */
a:hover { color: #00d4ff; }

/* Pseudo-class — first child */
li:first-child { font-weight: bold; }
💡

Use classes (not IDs) for most styling. IDs are very specific and harder to override. Classes can be reused across many elements.

03

Colors & Typography

color, font, text properties · ~6 min
✓ Done ▼
CSS
body {
  /* Colors — several formats */
  color: #c8d8f0;              /* hex */
  background-color: rgb(10,14,26); /* rgb */
  border-color: rgba(0,212,255,0.3); /* rgba (with opacity) */

  /* Typography */
  font-family: 'Segoe UI', Arial, sans-serif;
  font-size: 16px;           /* or rem, em, vw */
  font-weight: 400;           /* 400=normal, 700=bold */
  line-height: 1.7;           /* space between lines */
  letter-spacing: 1px;
}

h1 {
  font-size: clamp(2rem, 5vw, 3.5rem); /* responsive! */
  text-transform: uppercase;
  text-align: center;

  /* Gradient text */
  background: linear-gradient(90deg, #00d4ff, #00ff9d);
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
}
💡

Use rem for font sizes (relative to root) and clamp(min, ideal, max) for responsive text that scales smoothly between screen sizes.

04

The Box Model — Spacing & Sizing

margin, padding, border, width · ~7 min
✓ Done ▼

Every HTML element is a box. The CSS Box Model defines four areas: content → padding → border → margin. Understanding this fixes 90% of layout confusion.

CSS
.card {
  /* Content size */
  width: 300px;
  height: auto;

  /* Padding — space INSIDE the border */
  padding: 20px;               /* all sides */
  padding: 10px 20px;          /* top/bottom left/right */
  padding-top: 15px;

  /* Border */
  border: 1px solid rgba(0,212,255,0.3);
  border-radius: 10px;         /* rounded corners */

  /* Margin — space OUTSIDE the border */
  margin: 0 auto;              /* 0 top/bottom, auto left/right = centred */
  margin-bottom: 1.5rem;

  /* IMPORTANT: include border/padding in width */
  box-sizing: border-box;
}

/* Apply border-box to EVERYTHING (best practice) */
* { box-sizing: border-box; }
✅

Always put * { box-sizing: border-box; } at the top of your CSS. It makes width calculations predictable — padding and border no longer add to the element's size.

05

Flexbox — Modern Layout

display flex, align, justify · ~9 min
✓ Done ▼

Flexbox is a one-dimensional layout system — perfect for rows or columns. It makes centering, spacing, and alignment effortless.

CSS
/* Make a container flex */
.nav {
  display: flex;
  align-items: center;      /* vertical alignment */
  justify-content: space-between; /* horizontal spacing */
  gap: 1rem;                /* space between items */
  flex-wrap: wrap;          /* wrap to next line if needed */
}

/* justify-content values: flex-start | center | flex-end | space-between | space-around */
/* align-items values:    flex-start | center | flex-end | stretch */

/* Perfect centering — both horizontally AND vertically */
.hero {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}

/* Make items equal width */
.card { flex: 1; }  /* grow equally */
.sidebar { flex: 0 0 250px; }  /* fixed 250px */
💡

Flexbox works in one direction (row or column). For two-dimensional layouts (rows AND columns together), use CSS Grid — that's the next lesson!

06

CSS Grid — Two-Dimensional Layouts

grid-template, areas, fr unit · ~8 min
✓ Done ▼

CSS Grid lets you build two-dimensional layouts — both rows and columns simultaneously. It's perfect for card grids, page layouts and dashboards.

CSS
/* 3-column card grid */
.cards-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  /* 3 equal columns */
  gap: 1.5rem;
}

/* Responsive grid — auto-fill */
.auto-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 1.5rem;
}

/* Page layout with sidebar */
.page {
  display: grid;
  grid-template-columns: 250px 1fr;    /* sidebar | main */
  grid-template-rows: 60px 1fr 80px;    /* nav | content | footer */
  min-height: 100vh;
}

/* Make an item span multiple columns */
.featured-card { grid-column: 1 / -1; }  /* full width */

07

Responsive Design — Mobile First

@media queries, breakpoints · ~7 min
✓ Done ▼

Responsive design makes your website look good on all screen sizes — mobile, tablet and desktop. Use @media queries to apply different styles at different widths.

CSS
/* Default styles — mobile first (small screens) */
.grid {
  display: grid;
  grid-template-columns: 1fr;   /* single column on mobile */
  gap: 1rem;
}

/* Tablet — 768px and up */
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
  .nav-links { display: flex; }
}

/* Desktop — 1024px and up */
@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* Hide on mobile */
@media (max-width: 768px) {
  .sidebar { display: none; }
  .hamburger { display: flex; }
  section { padding: 60px 1rem; }
}
✅

Mobile first means writing your default styles for mobile, then adding @media (min-width:...) for larger screens. This is the professional approach — Google also ranks mobile-friendly sites higher.

08

Transitions & Animations

transition, @keyframes, transform · ~8 min
✓ Done ▼

CSS animations make your site feel alive and professional. transition smoothly changes a property over time. @keyframes creates complex animations.

CSS
/* TRANSITIONS — smooth state changes */
.btn {
  background: #00d4ff;
  transition: all 0.25s ease;  /* property duration timing */
}
.btn:hover {
  background: #00ff9d;
  transform: translateY(-3px);  /* lift up on hover */
  box-shadow: 0 8px 20px rgba(0,212,255,0.3);
}

/* TRANSFORMS */
.card:hover {
  transform: scale(1.05);    /* scale up */
  transform: rotate(5deg);   /* rotate */
}

/* KEYFRAME ANIMATIONS */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50%      { transform: scale(1.05); }
}

.hero { animation: fadeIn 0.8s ease forwards; }
.logo { animation: pulse 2s ease-in-out infinite; }
